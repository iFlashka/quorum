# Architecture Decision Records (ADR)

Здесь живут записи о неочевидных архитектурных и технологических решениях. Формат:

```
NNNN-kebab-title.md
```

Где `NNNN` — порядковый номер с нулями (`0001`, `0002`, …). Никаких пропусков; если решение отозвано — статус меняется на `Superseded by NNNN`, файл не удаляется.

## Структура одного ADR

```markdown
# ADR-NNNN: Заголовок

- **Дата:** YYYY-MM-DD
- **Статус:** Proposed | Accepted | Deprecated | Superseded by ADR-XXXX

## Контекст
В чём проблема, какие были варианты.

## Решение
Что выбрали.

## Последствия
Что становится проще, что — сложнее. Что станет головной болью при отказе.
```

## Что НЕ становится ADR

- Очевидные технические выборы (например, использование `clsx` вместо собственной реализации classNames)
- Решения, явно зафиксированные в [PROJECT.md](../../PROJECT.md) — там source of truth
- Временные хаки внутри одного файла — пиши коммент в коде, не плоди ADR
