# ADR-0005: Tray, нотификации, окно — desktop-поведение фазы 3

- **Дата:** 2026-05-06
- **Статус:** Accepted (реализован в фазе 3)

## Контекст

Цель фазы 3 — превратить Quorum из «обычного окна» в полноценный desktop-resident клиент: чтобы он сидел в трее, корректно сохранял окно, мог автозапускаться, гудел нативными toast'ами при упоминании, и показывал сколько непрочитано.

В Tauri 2.x для этого есть готовые building blocks (tray в core с фичей `tray-icon`, плагины `notification` / `window-state` / `autostart`). Спорные точки — где провести границы и куда положить state.

## Решения

### 1. Close-to-tray как единственное поведение по дефолту

При закрытии главного окна перехватываем `WindowEvent::CloseRequested`, делаем `api.prevent_close()` + `window.hide()`. Quit — только через пункт «Выйти» в tray-меню (или `Ctrl+Q`/native shortcut в фазе 4+ через global-shortcut).

**Why.** Discord-аналог; чат-приложение должно оставаться доступно. Пользователь намерен закрыть окно — почти всегда «спрятать», а не «выйти полностью», потому что иначе перестанут работать нотификации.

**Альтернативы отклонены.**

- *Опция в настройках «закрытие = выход».* Лишний выбор без явной причины. Если когда-то понадобится — добавим в фазе 7.
- *Разное поведение на macOS (cmd-Q quit, X hide).* На macOS native pattern другой, но фаза-приоритет Windows. Кросс-платформ-shadow оставлен на фазу 7.

### 2. Mute-state — источник истины в Rust, зеркало в zustand-сторе

Tray-меню содержит `CheckMenuItem` «Без уведомлений». Тот же toggle есть в UserCardMenu во фронте. Чтобы они не разъезжались:

- Источник истины — `TrayState.muted: Mutex<bool>` в Rust.
- При клике в tray: Rust меняет state и эмитит событие `tray://mute-toggled`. Фронт ловит, обновляет zustand + persistent store.
- При клике во фронте: zustand → `invoke('set_mute_state', { muted })` → Rust обновляет state + меняет `CheckMenuItem.set_checked()` чтобы tray-меню не врало.
- При старте: фронт читает persistent store и зеркалит обратно в Rust.

**Why.** Tray-меню всегда видно (пользователь может кликнуть когда окно закрыто) — оно не может ждать React. Зеркало в zustand нужно UI dropdown'у. Persistent store — потому что между перезапусками юзер ожидает что mute сохранён.

### 3. Mention-нотификации — фильтрация в одном месте

Вся логика «когда показывать toast» в `lib/notifications.ts`. Realtime-bridge не знает про окно/фокус/mute — он только зовёт hook на каждый `message.create`. Фильтр:

1. Не своё сообщение.
2. `mentionedUserIds` содержит meId (DM появятся в фазе 4+ — расширим тогда).
3. Не muted.
4. Окно невидимо или не сфокусировано (если юзер уже видит сообщение — toast лишний).
5. Permission получен (запрашиваем лениво при первом mention).

**Альтернативы отклонены.**

- *Notify на любое сообщение в активном канале.* Слишком шумно для группы из 5–10 человек; такое норм только в личных DM.
- *Notify на reply-to-me.* Откладываю — нужен server-side флаг или клиент должен помнить мои отправленные `messageId`. Добавлю в фазе 4 одновременно с DM.

### 4. Unread-бейдж — два видимых места

1. **Tray-иконка.** Tauri 2.x не имеет API «нарисовать бейдж поверх tray-icon» — только полная подмена. Готовлю две PNG: основная и `tray-unread.png` (с красной точкой). При count > 0 → ставим `tray-unread.png`, иначе обычная. Tooltip также меняется (`Quorum • N непрочитанных`).
2. **Window title.** `setTitle('Quorum • N')` — Windows автоматически отражает в taskbar.

**Why такой компромисс.** Полноценный native badge на overlay-icon (Windows `ITaskbarList3::SetOverlayIcon`) Tauri не экспонирует. Подмена tray-иконки даёт визуальное ощущение бейджа на 90% качества за ноль дополнительной инженерной работы. Window title — самый дешёвый способ сделать то же на taskbar.

### 5. Autostart — стартует с `--minimized`, идёт сразу в трей

Плагин `autostart` регистрирует бинарь с аргументом `--minimized`. В `lib.rs::setup` проверяем `std::env::args()`: если флаг есть → сразу `window.hide()`. Юзер из трея потом покажет окно когда нужно.

**Why.** Иначе autostart смысла не имеет: каждое утро всплывающее окно — антипаттерн.

### 6. Window-state — берём готовый плагин tauri-plugin-window-state без кастома

Сохраняет позицию/размер/maximized в JSON в app-config-dir. Без отдельной интеграции с UI.

## Что осталось на фазу 7

- Per-channel/per-guild mute (сейчас только глобальный).
- Click-через-toast → конкретный канал (Tauri 2.x ограничение, плюс нет универсального API).
- macOS-специфическое close-vs-quit поведение.
- Полноценный Settings-screen (mute, autostart переедут туда из dropdown'а).
- Кастомный native badge на window-icon (overlay-icon API через Manifest или прямые WinAPI).
